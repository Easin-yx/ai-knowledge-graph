# docs · 作者参考资料

这里收录的是**原作者构建本知识图谱时的过程文档**，不属于运行代码，删掉也不影响项目构建与部署。

之所以保留，是因为本项目定位是一个**「用 AI 维护的可视化知识图谱模板」**——这些文档正好示范了「如何用 AI 系统化地把一个领域整理成图谱数据」，对想要 fork 后做自己图谱的人是参考而非噪音。

## 内容

| 路径 | 说明 |
|---|---|
| `PRD.md` | 项目的产品需求文档，记录初始设计意图与范围 |
| `build-notes/loop-engineering.md` | 用 AI 迭代构建图谱的工作流（方法论） |
| `build-notes/<领域>/taxonomy.md` | 该领域的知识分类大纲（先有大纲，再填节点） |
| `build-notes/<领域>/progress.md` | 该领域图谱的整理进度记录 |
| `build-notes/<领域>/low_confidence.md` | 整理过程中不确定、待复核的条目 |
| `build-notes/black-myth.md` | 《黑神话：悟空》双视角拆解的源笔记 |

## 如果你是 fork 本模板来做自己图谱的人

你可以：

- **直接删掉整个 `docs/` 目录**——它只是示例，不影响项目运行；
- 或者**保留并仿照** `build-notes/` 的结构，记录你自己领域的 taxonomy / progress，把它当作你和 AI 协作的「工作台账」。
