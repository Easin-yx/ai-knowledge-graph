## L-1 · 「对比」边语义存疑：解释/编译 ↔ 同步/异步

- 边：`interpreter_vs_compiler__对比__sync_vs_async`（label「对比」，无向）
- 存疑原因：`interpreter_vs_compiler`（解释型 vs 编译型，语言**执行方式**）与 `sync_vs_async`（同步 vs 异步，程序**执行时序**）属于两个**不相关维度**；两节点 notes 均未解释为何要并列对比，疑似「为连接而连接」的硬凑边。
- 为何不擅自删：也可能是作者有意把「二分法概念」横向串联，属设计取舍；删边触及作者意图，按 Harness「禁止硬凑关系」反向同样需谨慎。
- 待人工裁决：
  1. 保留该边 → 建议在任一节点 notes 补一句对比理由（如「都是入门常遇到的『X vs Y』取舍」）。
  2. 删除该边 → 若认为二者无教学关联，移除以提升逻辑缜密性。
